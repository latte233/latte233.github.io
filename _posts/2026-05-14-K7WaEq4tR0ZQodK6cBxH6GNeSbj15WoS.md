## Claude 4 正式发布：编程天花板又刷新了

Anthropic 推出 Claude Opus 4 和 Sonnet 4。Opus 4 在 SWE-bench 得分 72.5%，能连续稳定运行数小时完成复杂长任务——比如7小时的开源代码重构。代理任务中捷径使用率降低65%，可靠性大幅提升。

同期发布 Claude Code 桌面端正式版，支持 VS Code/JetBrains 深度集成和 GitHub Actions 自动化。

AI编程的天花板又被抬了一截，复杂工程任务可以放心交给 Claude 了。

## Anthropic 开发者大会：AI编程从「补全」升级为「自主」

「Code w/ Claude」大会核心信号：企业客户AI生成代码比例一年内从15-20%跳到75%，Cursor内部30%的PR完全由AI独立完成。

Anthropic 提出「任务视界」概念——模型能脱离人类干预持续工作的时间，去年只有几分钟，现在已是数小时。

同步编程退场，异步开发流成新默认。

## 中国AI Agent市场：政策首次写入「智能体」

2025年中国AI Agent市场规模182.34亿元，同比增78.03%。2026年政府工作报告首次写入「智能体」，国务院设定2027年普及率超70%、2030年超90%的目标。

腾讯、阿里、字节全栈布局，垂直领域已有多个标杆项目落地（石化、政务、金融）。

政策背书+市场验证，B端Agent赛道窗口期明确。

## 智谱AI：编程API涨83%，调用量仍涨400%

智谱2025年营收7.24亿元，同比增131.9%。GLM Coding Plan付费开发者突破24.2万，Token调用量6个月涨15倍。GLM-5.1编程能力登顶开源模型榜首。

有意思的是API价格一季度累计涨83%，但调用量仍增长400%——需求刚性。

国内做AI编程工具和Agent开发的，GLM生态值得关注。

## LLM多轮对话「失忆」原因被找到

arXiv新研究揭示LLM在长对话中丢失指令的根本机制——「注意力闭锁」（Attention Closure），目标相关token随对话进行逐渐无法被注意力机制访问。

研究提出GAR（Goal Accessibility Ratio）度量，可预测模型何时开始「跑偏」，对部署长时程Agent系统有直接指导意义。

如果你在构建需要多轮上下文保持的Agent，这个研究值得细读。
