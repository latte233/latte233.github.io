本周 AI 行业从"密集发布"转向"深度消化"。GPT-5.6 Sol 仍处于受限预览阶段（仅约 20 家受信伙伴可访问），但 7 月将在 Cerebras 平台上线加速版（750 token/s），全面开放时间待定。DeepSeek 本周连放两招：6月27日联合北大开源 [DSpark](https://github.com/deepseek-ai/DeepSpec) 推理加速框架（单用户速度提升 60-85%），同时宣布 V4 正式版将于 7月中旬上线并引入峰谷定价机制——高峰时段 API 价格翻倍，这是开源模型厂商首次系统性引入动态定价。开源 Coding 模型赛道迎来新选手 [Ornith-1.0](https://github.com/deepreinforce-ai/Ornith-1)，MIT 协议、9B-397B 四种规格、自改进 RL 训练框架，在 TerminalBench 等基准上达到同规模开源模型 SOTA。

## 重点信号详解

### DSpark 推理加速框架

**象限**：Frameworks & Infrastructure
**环位**：Trial

**是什么**：DeepSeek 联合北京大学于 6月27日开源的推理加速框架，采用半自回归投机解码 + 置信度调度验证机制。在 DeepSeek-V4-Pro/Flash 生产环境中，单用户生成速度提升 60-85%，高并发场景吞吐量最高提升 661%，输出质量与目标模型完全一致。

**为什么重要**：推理成本是当前 LLM 工程化的核心瓶颈——Uber 因 Token-burning Agent 超支年度预算、GitHub Copilot Token 计费导致用户成本飙升 10-50 倍。DSpark 证明了纯算法优化可以在不增加硬件投入的前提下大幅提升推理效率，且已在大规模生产环境中验证。这对所有使用 DeepSeek-V4 或类似 MoE 架构的团队直接可用。

**当前状态**：已开源（MIT 协议），包含模型检查点和 DeepSpec 训练框架。已部署于 DeepSeek-V4 预览版服务引擎，经过真实线上流量验证。论文由创始人梁文锋署名。

**建议下一步**：如果团队使用 DeepSeek-V4 或计划在 7月中旬 V4 正式版上线后接入，建议在测试环境中评估 DSpark 加速效果。关注 V4 正式版的峰谷定价策略对总体成本的影响。

### Ornith-1.0 开源 Coding 模型

**象限**：Models & Research
**环位**：Trial

**是什么**：DeepReinforce 于 6月25日发布的开源 Coding 模型族，包含 9B Dense、31B Dense、35B MoE 和 397B MoE 四种规格，基于 Gemma 4 和 Qwen 3.5 预训练权重后训练。核心创新是"自改进 RL 脚手架"——模型联合优化任务脚手架（工具链构建）和解决方案（代码生成），在 TerminalBench 2.1、SWE-Bench、NL2Repo 等基准上达到同规模开源模型 SOTA。

**为什么重要**：开源 Coding 模型赛道此前主要由 DeepSeek-Coder 和 Qwen-Coder 主导，Ornith-1.0 引入了新的训练范式（自改进 RL），且 MIT 协议下完全开源。397B MoE 版本在部分基准上接近闭源模型表现，9B 版本适合本地部署——为不同规模团队提供了可选方案。

**当前状态**：MIT 协议开源，权重发布在 HuggingFace。支持 vLLM、SGLang、Transformers、llama.cpp 多框架部署。已与 OpenHands、OpenCode、Hermes Agent 集成。Simon Willison 已撰文关注。

**建议下一步**：用 9B 版本在本地环境做 Coding 能力快速验证，对比当前使用的模型（如 DeepSeek-Coder 或 Qwen-Coder）。如果团队有 Agent Coding 需求，可评估 35B MoE 版本在工具调用和多步规划上的表现。

### MCP 安全危机与 NSA 加固指南

**象限**：Practices & Ecosystem
**环位**：Trial

**是什么**：6月 MCP 安全问题集中爆发——BlueRock Security 披露 MarkItDown MCP 服务器存在 fURI 漏洞（SSRF → 云基础设施接管）；Adversa AI 统计显示 12,520 个暴露的 MCP 服务器中约 40% 无认证、67 个 CVE 已被分配、仅 8.5% 使用 OAuth；NSA 于 5月20日发布 MCP 安全设计指南（9 项最低要求），要求联邦承包商在 4 个月内完成整改。

**为什么重要**：MCP 是当前 AI Agent 生态的标准连接协议，但其安全设计先天不足——任意 URI 调用、无强制认证、跨 Agent 内容传播漏洞。MarkItDown 作为 GitHub 飙星榜第一的项目，其 MCP 服务器漏洞意味着大量用户可能面临云密钥泄露风险。NSA 的介入标志着 MCP 安全从社区讨论升级为国家监管层面。

**当前状态**：NSA 指南已发布，但 MCP 协议层面的修复尚未到来。BlueRock、OX Security 等安全公司持续披露新漏洞。社区讨论活跃但缺乏统一的安全基线实施。

**建议下一步**：立即审计团队使用的所有 MCP 服务器配置——确保不在公网暴露、启用认证、限制出站网络访问。如果使用 MarkItDown MCP 服务器，确保不在 EC2 或其他云实例上运行，或等待官方修复。建议参考 NSA 指南的 9 项要求制定内部 MCP 安全规范。

### GPT-5.6 Sol 受限预览持续

**象限**：Models & Research
**环位**：Trial

**是什么**：OpenAI 6月26日发布的 GPT-5.6 系列旗舰模型 Sol 仍处于受限预览阶段，仅约 20 家受信合作伙伴可通过 API 和 Codex 访问。Sol 在 TerminalBench 2.1 以 91.9% 登顶，CTF 命中率 96.7%。计划 7 月在 Cerebras 平台上线加速版（750 token/s），但全面开放时间受美国政府审查影响尚未确定。

**为什么重要**：GPT-5.6 Sol 的编程能力超越 Claude Mythos 5，但受限预览意味着大部分开发者无法直接使用。7 月 Cerebras 加速版的上线可能扩大访问范围，但政府审查的不确定性仍在。三级定价体系（Sol $5/$30、Terra $2.50/$15、Luna $1/$6）一旦全面开放，将直接冲击 Claude Opus 4.8 的定价。

**当前状态**：受限预览中。Terra 作为 GPT-5.5 平替方案（性能相当、价格减半）是当前最实用的选项。完整评估结果将在模型广泛可用时公布。

**建议下一步**：无需立即行动。关注 7 月 Cerebras 加速版上线后的访问范围扩展。如果团队有 Codex 访问权限，可尝试用 Terra 替代 GPT-5.5 做成本对比。
